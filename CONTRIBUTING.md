# Contributing to Rudra Prajapati's Portfolio

Thank you for your interest in contributing! Whether you are reporting a bug, proposing new features, or submitting pull requests, your help is welcome.

## 🚀 How to Contribute

### 1. Reporting Bugs
- Search existing issues to ensure the bug hasn't been reported.
- If it hasn't, open a new issue detailing:
  - Clear steps to reproduce the issue.
  - Expected vs. actual behavior.
  - Screen captures or error logs where applicable.

### 2. Suggesting Improvements
- Open a new issue or discussion thread outlining your idea.
- Explain the user benefit and proposed design or flow.

### 3. Submitting Pull Requests
1. Fork this repository.
2. Create a new topic branch:
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. Make your edits. Ensure you adhere to:
   - TypeScript safety (run `npm run lint` and `npm run build` locally).
   - Responsive design parameters.
4. Commit your changes with a clear commit description:
   ```bash
   git commit -m "Add feature: descriptions here"
   ```
5. Push to your branch and open a Pull Request against the `main` branch.

## 🎨 Design System Constraints
- Keep to the warm beige aesthetic (`bg-warm-beige` / `#F9F6F0`).
- Ensure custom mouse actions and transitions run smoothly on standard viewports.
- Adhere to semantic HTML structure and descriptive metadata configurations.
